# Tarea 0 | Entrega Final

## Proyecto

El objetivo de esta entrega es familiarizarse con el uso de Git e IntelliJ, por lo que se
le solicitará cumplir con una serie de requisitos.

Recuerde que para acceder a su proyecto desde su computadora debe clonar el repositorio que contiene su usuario de Github, obtenido mediante el enlace publicado
en U-Cursos.

El archivo ``Tarea0`` con el que trabajará se encuentra en la ubicación ``src/main/scala/model``.
- Cree una función llamada ``greet`` en el archivo ``Tarea0``, la cual recibe como argumento un nombre e imprime un saludo al nombre utilizado como parámetro.
- Cree un archivo ``Main`` que le permita preguntar por su nombre al usuario y se salude al mismo con la función ``greet``. Esto debiese poder ejecutarse.

## Git

El trabajo realizado para esta entrega se debe realizar en una nueva rama que deberá
llamar ``entrega-final-0``:

1. Puede usar el comando ``git branch`` en todo momento para saber en que rama se encuentra, actualmente su única rama debiese ser ``master``, por lo que se encontrará allí.

2. Utilice el comando ``git branch <branch_name>`` para crear la rama. En este caso, sería
``git branch entrega-final-0``.

3. Recuerde que para que su progreso pueda ser almacenado en dicha rama, debe cambiarse de
rama utilizando el comando ``git checkout <branch_name>``. En este caso, sería
``git checkout entrega-final-0``.

*Tenga en cuenta que el cuerpo docente tiene acceso total a su repositorio.*

## Entrega

Para subir su entrega, deberá crear un *pull request* desde la rama
``entrega-final-0`` a la rama ``main`` llamado ``Tarea 0 - Entrega Final``.

Es importante que **no hagan merge** de la rama ``entrega-final-0`` a la rama ``main`` 
para que el cuerpo docente pueda revisar su *pull request*.

Por *U-Cursos* debe entregar un único archivo llamado ``entrega-final-0.txt`` con el
siguiente contenido:

```
Nombre: <Nombre completo>
Pull Request: <Link del pull request>
```

No cumplir con el formato pedido de una entrega podría llevar a no ser
considerada, y para una Entrega Final, tiene descuentos en su nota final.

## Días de emergencia

Usted cuenta con 7 días de emergencia durante el semestre. Estos pueden ser utilizados en cada *Entrega Final*, y funcionan como una posibilidad para entregar su tarea con atraso.

Puede utilizar máximo 3 días de atraso por tarea. Si no cuenta con más días de atraso y entrega luego del plazo establecido, se considerará el último commit dentro del período de entrega válido.
